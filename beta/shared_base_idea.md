# Shared Base Config — Design Notes

This is a draft concept for refactoring the 4 ESPHome YAML files into a single shared base config. Not implemented yet — see ENHANCEMENTS.md for context.

## The Problem

All 4 toolhead YAML files (T0–T3) are nearly identical. Any change to shared logic — like the LED effects we recently added — has to be manually applied to all 4 files. This is error-prone and gets worse as the project grows.

## The Solution

ESPHome supports `substitutions` and `packages: !include` which lets us split the config into:

- **`base.yaml`** — all shared logic (LED, MQTT handlers, PN532, I2C). Never edited directly.
- **`toolhead-tX.yaml`** — thin wrapper per toolhead. Only contains the device name, static IP, and a `substitutions` block defining the toolhead ID.

The `${toolhead_id}` substitution variable gets replaced at compile time, so MQTT topics, log messages, and LED names all resolve correctly for each toolhead without any duplicated logic.

## Dependency

This refactor should be done **before** building the install script, since the install script will generate the thin `toolhead-tX.yaml` wrappers automatically based on how many toolheads the user configures.

## base.yaml

```yaml
# =============================================================
# NFC Toolchanger Spoolman — Shared Base Config
# =============================================================
# Do not edit this file directly. All toolhead-specific values
# (name, IP, MQTT topics) are set in the individual toolhead files.
# =============================================================

# ESP32-S3 board configuration
esp32:
  variant: esp32s3
  framework:
    type: esp-idf

logger:
api:

ota:
  - platform: esphome

captive_portal:

# I2C bus for PN532
i2c:
  sda: GPIO1
  scl: GPIO2
  frequency: 10000
  scan: true

# Onboard WS2812 RGB LED (GPIO21 on Waveshare ESP32-S3-Zero)
light:
  - platform: esp32_rmt_led_strip
    rgb_order: RGB
    pin: GPIO21
    num_leds: 1
    name: "${friendly_name} Status LED"
    id: status_led
    default_transition_length: 0s
    effects:
      - pulse:
          name: "Low Spool Warning"
          transition_length: 1s
          update_interval: 1s
          min_brightness: 10%
          max_brightness: 80%

# MQTT subscriptions — topics use ${toolhead_id} substitution
mqtt:
  broker: !secret mqtt_broker
  username: !secret mqtt_username
  password: !secret mqtt_password
  on_message:
    - topic: "nfc/toolhead/${toolhead_id}/color"
      then:
        - if:
            condition:
              lambda: 'return x == "error";'
            then:
              # Flash red 3 times — unknown or unregistered tag
              - light.turn_on:
                  id: status_led
                  red: 1.0
                  green: 0.0
                  blue: 0.0
                  brightness: 100%
                  effect: none
              - delay: 200ms
              - light.turn_off:
                  id: status_led
              - delay: 150ms
              - light.turn_on:
                  id: status_led
                  red: 1.0
                  green: 0.0
                  blue: 0.0
                  brightness: 100%
              - delay: 200ms
              - light.turn_off:
                  id: status_led
              - delay: 150ms
              - light.turn_on:
                  id: status_led
                  red: 1.0
                  green: 0.0
                  blue: 0.0
                  brightness: 100%
              - delay: 200ms
              - light.turn_off:
                  id: status_led
            else:
              - lambda: |-
                  std::string hex = x;
                  if (hex.length() == 6) {
                    int r = stoi(hex.substr(0, 2), nullptr, 16);
                    int g = stoi(hex.substr(2, 2), nullptr, 16);
                    int b = stoi(hex.substr(4, 2), nullptr, 16);
                    auto call = id(status_led).turn_on();
                    call.set_effect("none");
                    call.set_rgb(r / 255.0, g / 255.0, b / 255.0);
                    call.perform();
                  } else if (hex == "000000") {
                    id(status_led).turn_off().perform();
                  }
    - topic: "nfc/toolhead/${toolhead_id}/low_spool"
      then:
        - if:
            condition:
              lambda: 'return x == "true";'
            then:
              # Start breathing effect — spool is at or below LOW_SPOOL_THRESHOLD
              - light.turn_on:
                  id: status_led
                  effect: "Low Spool Warning"
            else:
              # Clear breathing effect — restore solid colour
              - light.turn_on:
                  id: status_led
                  effect: none

# PN532 NFC reader
pn532_i2c:
  id: pn532_board
  on_tag:
    then:
      - logger.log:
          format: "Tag scanned on ${toolhead_id}: %s"
          args: [x.c_str()]
      # Flash white 3x on successful scan
      - light.turn_on:
          id: status_led
          red: 1.0
          green: 1.0
          blue: 1.0
          brightness: 100%
      - delay: 150ms
      - light.turn_off:
          id: status_led
      - delay: 100ms
      - light.turn_on:
          id: status_led
          red: 1.0
          green: 1.0
          blue: 1.0
          brightness: 100%
      - delay: 150ms
      - light.turn_off:
          id: status_led
      - delay: 100ms
      - light.turn_on:
          id: status_led
          red: 1.0
          green: 1.0
          blue: 1.0
          brightness: 100%
      - delay: 150ms
      - light.turn_off:
          id: status_led
      - mqtt.publish:
          topic: "nfc/toolhead/${toolhead_id}"
          payload: !lambda |-
            return "{\"uid\": \"" + x + "\", \"toolhead\": \"${toolhead_id}\"}";
```

## toolhead-t0.yaml (and T1, T2, T3)

```yaml
# =============================================================
# Toolhead T0 — edit this file for T0-specific settings
# All shared logic lives in base.yaml
# =============================================================

substitutions:
  toolhead_id: "T0"
  friendly_name: "Toolhead T0 NFC"

esphome:
  name: toolhead-t0
  friendly_name: ${friendly_name}
  min_version: 2025.11.0
  name_add_mac_suffix: false

packages:
  base: !include base.yaml

wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password
  manual_ip:
    static_ip: 192.168.X.X
    gateway: 192.168.X.1
    subnet: 255.255.255.0
  ap:
    ssid: "Toolhead-T0 Fallback"
    password: "CHANGE_ME"
```

T1, T2, T3 are identical — just change `toolhead_id`, `friendly_name`, `name`, and `static_ip`.

## What the install script generates

When a user runs `./install.sh` and answers `4` for number of toolheads, the script generates:

- `esphome/toolhead-t0.yaml`
- `esphome/toolhead-t1.yaml`
- `esphome/toolhead-t2.yaml`
- `esphome/toolhead-t3.yaml`

Each with the correct `toolhead_id` substitution and the static IP the user provided for that toolhead. `base.yaml` is never touched by the install script.
